## About Delivery Plans

Delivery plans outline the implementation roadmap for significant features or initiatives. Each delivery plan should include:

- **Overview**: Feature description and business value
- **Scope**: What's included and what's not
- **Dependencies**: Technical and organizational dependencies
- **Phases**: Breakdown of implementation phases
- **Timeline**: Estimated dates and milestones
- **Resources**: Team members and skill requirements
- **Risks**: Identified risks and mitigation strategies
- **Success Criteria**: How success will be measured

## Creating a New Delivery Plan

Delivery plans follow the naming convention:
`{feature-name}-delivery-plan-v{X.Y}.md`
