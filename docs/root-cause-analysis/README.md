## About RCA Documents

Root Cause Analysis documents provide in-depth investigation of major incidents. Each RCA should include:

- **Incident Summary**: What happened
- **Timeline**: Detailed chronology of events
- **Root Cause**: Identified root cause(s)
- **Contributing Factors**: Other factors that contributed
- **Impact Assessment**: Scope and severity of impact
- **Resolution**: How the incident was resolved
- **Prevention**: Measures to prevent recurrence
- **Action Items**: Follow-up tasks with owners

## Creating a New RCA

RCAs are typically created after:
- Production outages
- Data loss incidents
- Security breaches
- Major performance degradations

RCAs follow the naming convention:
`{incident-name}-analysis.md`
