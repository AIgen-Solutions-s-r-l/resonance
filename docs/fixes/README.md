## About Fix Documents

Fix documents provide concise documentation of bug fixes. Each fix document should include:

- **Problem**: Concise problem statement
- **Root Cause**: Brief root cause from investigation
- **Solution**: Implemented fix
- **Changes**: Modified files and descriptions
- **Testing**: Test cases added or verified
- **Verification**: How to verify the fix
- **Prevention**: Measures to prevent recurrence

## Creating a New Fix Document

Use the template in `docs/templates/fix-template.md` to create a new fix document.

Fix documents follow the naming convention:
`{issue-name}-fix-{YYYYMMDD}.md`
